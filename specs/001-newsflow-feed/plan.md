## Summary

Build NewsFlow: a React + Vite + TypeScript SPA that displays categorized Indian news across
five tabs, fetching Google News RSS feeds client-side via the allorigins.win CORS proxy and
parsing them with `rss-parser`. State is managed with React Context (active category) and
local hook state (per-feed articles/status). The UI is mobile-first with Tailwind CSS,
features a sticky tab bar, relative timestamps via `date-fns`, infinite scroll via
IntersectionObserver, and 10-minute auto-refresh. Deployed to Vercel as a static SPA.

---

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode), Node 20 LTS
**Primary Dependencies**: React 18, Vite 5, Tailwind CSS 3, rss-parser 3.x, date-fns 3.x
**Storage**: In-memory only (React state, no persistence, no IndexedDB)
**Testing**: None in this iteration (not requested in spec)
**Target Platform**: Modern mobile and desktop browsers; deployed on Vercel free tier
**Project Type**: Single-page web application (SPA)
**Performance Goals**: First article visible ≤5s on mobile; tab switch ≤1s (SC-001, SC-002)
**Constraints**: allorigins.win free tier (no auth, no rate limit docs); Google News RSS
  returns ~10–20 items per feed; no pagination parameters available on Google News RSS
**Scale/Scope**: Single user, client-side only, read-only news feed

## Phase 0: Research

**Status**: ✅ Complete — see [research.md](research.md)

Key decisions made:
1. **CORS proxy**: allorigins.win in both dev and prod; Vite dev server proxies
   `/allorigins` → `https://api.allorigins.win` for identical parsing in both envs
2. **Parsing**: `rss-parser.parseString(contents)` with custom `source` field config
3. **Infinite scroll**: IntersectionObserver sentinel + `displayCount` window over
   pre-fetched articles (RSS has no pagination — simulate by revealing items)
4. **Auto-refresh**: `setInterval` in `useFeed`, cleanup on unmount/category change
5. **Abort**: `AbortController` per fetch, cleanup in `useEffect` return
6. **Relative time**: `date-fns formatDistanceToNow`

### User Story 3 — Auto-Refresh (P3)

**Goal**: Feed silently refreshes every 10 minutes.

**Implementation**: Already wired into `useFeed.ts` (setInterval + deduplication).
No additional files needed — verify behaviour in the browser by temporarily reducing
`REFRESH_INTERVAL_MS` to 10s during manual testing.

**Checkpoint**: Feed updates without interaction; no scroll reset.

---

### User Story 4 — Infinite Scroll (P4)

**Goal**: Articles reveal progressively; end-of-feed message shown.

**Files**:
18. `src/hooks/useIntersectionObserver.ts`

    wire `loadMore` from `useFeed`

**Checkpoint**: Scrolling to bottom reveals next PAGE_SIZE articles; "You're all caught up" shows at end.
