## Summary

Add a bookmarks feature that lets users save articles with a single tap, persist them across sessions via `localStorage`, and view/remove them from a dedicated Bookmarks tab. Implemented entirely client-side with no new npm dependencies: a new `BookmarkContext` at the App root, a `BookmarksContainer` component, a restructured `NewsCard` layout (bookmark button as sibling of the article `<a>`), and a new `ActiveTab` union type that extends `CategoryId` to include `'bookmarks'`.

## Implementation Notes

### `src/types/index.ts`
- Add `export type ActiveTab = CategoryId | 'bookmarks'`
- Add `export interface BookmarkedArticle extends NewsArticle { savedAt: number }`
- Update `CategoryContextValue`: `activeCategory: ActiveTab`, `setActiveCategory: (id: ActiveTab) => void`

### `src/context/CategoryContext.tsx`
- Change internal `useState<CategoryId>` to `useState<ActiveTab>` (initial value `'top'` is still valid as `CategoryId ⊂ ActiveTab`)
- Update the context type annotation to `CategoryContextValue` (picks up the modified interface automatically)

### `src/context/BookmarkContext.tsx` *(new)*
- `BookmarkContextValue` interface: `{ bookmarks: BookmarkedArticle[], toggleBookmark: (article: NewsArticle) => void, isBookmarked: (articleId: string) => boolean }`
- Lazy `useState` initializer reads from `localStorage.getItem('newsflow_bookmarks')` wrapped in `try/catch` (returns `[]` on failure)
- `useEffect([bookmarks])` writes `JSON.stringify(bookmarks)` to localStorage, also in `try/catch`
- `toggleBookmark`: prepend on add (`[{ ...article, savedAt: Date.now() }, ...prev]`), filter-by-id on remove
- `isBookmarked`: `bookmarks.some(b => b.id === articleId)`
- `useBookmarkContext()` hook throws descriptive error if used outside `BookmarkProvider`

### `src/components/BookmarksContainer/BookmarksContainer.tsx` *(new)*
- No props; reads `bookmarks` and `toggleBookmark` from `useBookmarkContext()`
- Empty state: `<p>No bookmarks yet. Tap the bookmark icon on any article to save it.</p>`
- Filled state: scrollable list of `<NewsCard article={bookmark} />` (array is already most-recent-first)
- No `useFeed`, no category, no `ScrollSentinel` — renders entirely from localStorage state

### `src/components/TabBar/TabBar.tsx`
- After the 5 category tab buttons, add a hardcoded Bookmarks tab:
  ```tsx
  <button onClick={() => setActiveCategory('bookmarks')} aria-current={activeCategory === 'bookmarks' ? 'page' : undefined} ...>
    Bookmarks
  </button>
  ```
- Same active/inactive Tailwind classes as category tabs

### `src/components/NewsCard/NewsCard.tsx`
- Wrap existing content in `<div className="relative border-b border-gray-100">`
- Move `border-b` from `<a>` to the outer wrapper (avoid duplicate border)
- Add `pr-14` to the inner content container inside `<a>` so text doesn't overlap the button
- Add `<button aria-label="Bookmark article" / "Remove bookmark" onClick={...}>` as sibling of `<a>`; absolutely positioned `top-2 right-2`, min `w-11 h-11` touch target
- Consume `useBookmarkContext()` internally for `isBookmarked(article.id)` and `toggleBookmark(article)`
- Use filled vs outline bookmark icon (SVG or Unicode `🔖` / outline equivalent)

### `src/App.tsx`
- Add `BookmarkProvider` import + `BookmarksContainer` import
- Add `MainView` helper component (reads `activeCategory`; returns `<BookmarksContainer />` or `<FeedContainer />`)
- Wrap tree: `<BookmarkProvider><CategoryProvider>…<MainView />…</CategoryProvider></BookmarkProvider>`

### `src/components/FeedContainer/FeedContainer.tsx`
- One-line change: `useFeed(activeCategory as CategoryId)` with inline comment explaining the safe cast
- No other changes needed

### `tests/e2e/bookmarks.spec.ts` *(new)*
- Mock RSS feed with 3 articles (reuse `makeArticles` helper)
- US1: bookmark icon visible, fills on tap, persists after reload, empties on second tap
- US2: Bookmarks tab in tab bar, saved articles appear (most-recent first), empty-state message
- US3: remove from feed updates icon, remove from bookmarks tab removes from list
- Edge: bookmark icon tap does not navigate; duplicate prevention; icon consistent across tabs
