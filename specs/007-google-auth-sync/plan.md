# Implementation Plan: Google Sign-In with Cloud Sync

**Branch**: `007-google-auth-sync` | **Date**: 2026-03-02 | **Spec**: `specs/007-google-auth-sync/spec.md`
**Input**: Feature specification from `/specs/007-google-auth-sync/spec.md`

## Summary

Add optional Google Sign-In via Firebase Auth. When signed in, bookmarks and category
preferences sync to Firestore in real time across devices. Signed-out users see zero change
in behaviour вҖ” local storage continues to be the source of truth. New `AuthContext` exposes
user state app-wide. `BookmarkContext` and `CategoryContext` are extended to dual-write to
Firestore when a user is authenticated.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Primary Dependencies**: `firebase@^10` (modular SDK вҖ” tree-shakeable)
**Storage**: Firestore (`users/{uid}/bookmarks/`, `users/{uid}/preferences`) + existing localStorage
**Testing**: Playwright (signed-out UI + injected mock state); no Firebase emulator required
**Target Platform**: SPA, Vercel free tier (static вҖ” all Firebase calls are client-side)
**Project Type**: Web application (client-side SPA)
**Performance Goals**: Auth state resolves < 1s on page load (Firebase SDK caches session in IndexedDB)
**Constraints**: Firebase Spark free tier; no server; no new state management library
**Scale/Scope**: Personal app; well within Firestore free tier (50k reads/day, 20k writes/day)

## Phase 0: Research

See `research.md`. All decisions resolved:
1. Firebase v10 modular SDK (tree-shakeable, ~150KB gzipped for auth+firestore)
2. Nested Firestore: `users/{uid}/bookmarks/` + single `users/{uid}/preferences` doc
3. Offline persistence via `persistentLocalCache` with `persistentMultipleTabManager`
4. `AuthProvider` wraps `BookmarkProvider` and `CategoryProvider`
5. First-sign-in localвҶ’cloud merge via `writeBatch`
6. Sign-out: unsubscribe Firestore listeners, revert to localStorage
7. Dual-write (localStorage + Firestore) for bookmarks and preferences when signed in
8. Playwright testing: signed-out UI tests + `window.__MOCK_AUTH_USER__` injection for signed-in UI

### Key Design Details

#### firebase.ts вҖ” graceful degradation when env vars missing

```ts
const configured = [
  import.meta.env.VITE_FIREBASE_API_KEY,
  import.meta.env.VITE_FIREBASE_AUTH_DOMAIN,
  import.meta.env.VITE_FIREBASE_PROJECT_ID,
  import.meta.env.VITE_FIREBASE_STORAGE_BUCKET,
  import.meta.env.VITE_FIREBASE_MESSAGING_SENDER_ID,
  import.meta.env.VITE_FIREBASE_APP_ID,
].every(Boolean)

export const firebaseApp = configured ? initializeApp(config) : null
export const firebaseAuth = configured ? getAuth(firebaseApp!) : null
export const firebaseDb = configured
  ? initializeFirestore(firebaseApp!, { localCache: persistentLocalCache({ tabManager: persistentMultipleTabManager() }) })
  : null
export const googleProvider = configured ? new GoogleAuthProvider() : null
```

#### AuthContext вҖ” mock injection for Playwright tests

```ts
// In dev mode only: allow Playwright to inject a signed-in user via addInitScript
function getInitialMockUser(): UserProfile | null {
  if (!import.meta.env.DEV) return null
  const w = window as { __MOCK_AUTH_USER__?: UserProfile }
  return w.__MOCK_AUTH_USER__ ?? null
}
```

#### BookmarkContext вҖ” listener lifecycle

- Signed out: initialise from localStorage, write to localStorage on toggle
- Signed in: set up `onSnapshot` on `users/{uid}/bookmarks`, mirror writes to localStorage for offline resilience
- On sign-in: run merge (local bookmarks not in cloud вҶ’ write to Firestore)
- On sign-out: unsubscribe, reload from localStorage
