## Summary

Add system-preference-aware dark mode with a manual toggle and localStorage persistence. Tailwind `darkMode: 'class'` applies all dark styles via a `dark` class on `<html>`. A blocking inline script in `index.html` sets this class before React mounts — preventing any flash of the wrong theme. A new `ThemeContext` at the App root manages state, syncs the DOM class, and listens for OS theme changes. Only `TabBar` consumes `ThemeContext`; all other components are themed passively via `dark:` utility prefixes. Zero new npm dependencies.

## Implementation Notes

### `index.html`
Add before `</head>`:
```html
<script>
  (function () {
    try {
      var t = localStorage.getItem('newsflow_theme');
      if (t === 'dark') { document.documentElement.classList.add('dark'); return; }
      if (t === 'light') { return; }
    } catch (e) {}
    if (window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches) {
      document.documentElement.classList.add('dark');
    }
  })();
</script>
```

### `tailwind.config.js`
Add `darkMode: 'class'` as the first key.

### `src/types/index.ts`
Add `export type Theme = 'light' | 'dark'`

### `src/context/ThemeContext.tsx` *(new)*
- Lazy `useState` initializer: reads `localStorage.getItem('newsflow_theme')`, falls back to `matchMedia`
- `useEffect([theme])`: syncs `document.documentElement.classList` — does NOT write to localStorage
- `useEffect([])`: registers `matchMedia.addEventListener('change', handler)` — handler only updates state when `localStorage.getItem('newsflow_theme') === null`; cleanup removes listener
- `toggleTheme()`: flips state AND writes to localStorage (only manual toggles persist)

### `src/App.tsx`
- Add `ThemeProvider` as outermost wrapper
- Add `dark:bg-gray-900` to root `<div>`
